## Modular by Design

Better-TS-Stack follows a **modular architecture**. Instead of generating a monolithic codebase, it composes your project from independent, well-designed modules that work together seamlessly.

Each module is:
- **Self-contained** — Has its own templates, configs, and logic
- **Optional** — Include only what you need
- **Composable** — Works with other modules automatically
- **Best-practice** — Follows industry standards

## How Modules Work Together

When you select multiple modules, Better-TS-Stack intelligently wires them together:

### Example: Full-Stack with Auth

If you select:
- Backend (Express)
- Frontend (Next.js)
- Database (PostgreSQL + Prisma)
- Auth (Yes)

The CLI automatically:
1. Creates Express auth routes with JWT
2. Generates Prisma User schema with password hashing
3. Sets up Next.js auth context and hooks
4. Configures environment variables for both sides
5. Adds protected route middleware

### Example: API with Docker

If you select:
- Backend (Express)
- Database (MongoDB)
- Docker (Yes)

The CLI automatically:
1. Generates Express API with Mongoose models
2. Creates Dockerfile with multi-stage build
3. Creates docker-compose.yml with MongoDB service
4. Configures connection strings for containerized environment

## Module Configuration

Each module contributes to the final `package.json` and configuration files:

```json
{
  "dependencies": {
    // Backend module
    "express": "^4.18.0",
    "cors": "^2.8.5",
    
    // Database module (Prisma)
    "@prisma/client": "^7.4.1",
    "@prisma/adapter-pg": "^7.4.1",
    
    // Auth module (Express)
    "jsonwebtoken": "^9.0.2",
    "bcrypt": "^5.1.1",

    // Auth module (Next.js)
    "better-auth": "^1.4.19"
  }
}
```

## Customizing Modules

After generation, you can extend any module:

<Callout type="tip">
  **All generated code is editable.** The modules provide a solid foundation, but you're free to modify, extend, or replace any part.
</Callout>

### Adding to Backend
- New routes in `src/routes/`
- New middleware in `src/middleware/`
- Additional controllers in `src/controllers/`

### Adding to Frontend
- New pages in `app/` directory
- New components in `components/`
- API routes in `app/api/`

### Adding to Database
- New models in Prisma schema
- New Mongoose schemas
- Additional migrations

## Module Dependencies

Some modules work better together:

| Combination | Benefits |
|-------------|----------|
| Backend + Database | Complete API with data persistence |
| Frontend + Backend | Full-stack with type-safe API calls |
| Backend + Auth + Database | Secure API with user management |
| Everything + Docker | Production-ready containerization |

<Callout type="info">
  The CLI handles all wiring automatically. Just select the modules you want, and it configures the connections.
</Callout>
