## Overview

The Database module adds database connectivity to your project. Selection happens in two steps during the CLI prompts:

1. **Database type** - `none`, `postgresql`, or `mongodb`
2. **ORM/ODM** - shown only when a database was selected

## Available Options

| Database   | ORM/ODM  | Available in         | Type safety                 |
| ---------- | -------- | -------------------- | --------------------------- |
| PostgreSQL | Prisma   | Backend + Full-stack | Full (auto-generated types) |
| PostgreSQL | Drizzle  | **Full-stack only**  | Full                        |
| MongoDB    | Mongoose | **Backend only**     | Partial                     |
| MongoDB    | Prisma   | Backend + Full-stack | Full (via connector)        |

<Callout type="info">
  Drizzle is only available for Next.js (full-stack) projects. Mongoose is only
  available for Express (backend) projects.
</Callout>

### Prisma client setup

<Tabs items={['Express', 'Next.js']}>
  <Tab value="Express">
    ```typescript
    // src/lib/prisma.ts
    import { PrismaClient } from "@prisma/client";

    const globalForPrisma = globalThis as unknown as {
      prisma: PrismaClient | undefined;
    };

    export const prisma =
      globalForPrisma.prisma ??
      new PrismaClient({
        log: process.env.NODE_ENV === "development"
          ? ["query", "error", "warn"]
          : ["error"],
      });

    if (process.env.NODE_ENV !== "production") {
      globalForPrisma.prisma = prisma;
    }
    ```

  </Tab>
  <Tab value="Next.js">
    ```typescript
    // lib/prisma.ts
    import { PrismaPg } from "@prisma/adapter-pg";
    import { PrismaClient } from "../lib/generated/client";

    const globalForPrisma = global as unknown as { prisma: PrismaClient };

    const adapter = new PrismaPg({
      connectionString: process.env.DATABASE_URL,
    });

    const prisma =
      globalForPrisma.prisma ||
      new PrismaClient({ adapter });

    if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = prisma;

    export default prisma;
    ```

  </Tab>
</Tabs>

## PostgreSQL + Drizzle _(Next.js only)_

### Generated files

```text
lib/
  schema.ts              # Table definitions
  db.ts                  # Drizzle database connection
drizzle.config.ts        # Drizzle Kit configuration
```

<Callout type="tip">
  If Better Auth is also selected, the generated `lib/schema.ts` contains Better
  Auth core tables (`user`, `session`, `account`, `verification`) instead of a
  sample table.
</Callout>

### schema.ts (base, without Better Auth)

```typescript
// lib/schema.ts
import { pgTable, text, timestamp } from "drizzle-orm/pg-core";

export const users = pgTable("users", {
  id: text("id").primaryKey(),
  name: text("name").notNull(),
  email: text("email").notNull().unique(),
  createdAt: timestamp("created_at").defaultNow(),
});
```

### Database connection

```typescript
// lib/db.ts
import { drizzle } from "drizzle-orm/node-postgres";
import { Pool } from "pg";

import * as schema from "./schema";

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
});

export const db = drizzle(pool, { schema });
```

### Commands

```bash
npm run db:generate   # drizzle-kit generate
npm run db:migrate    # drizzle-kit migrate
npm run db:studio     # drizzle-kit studio
```

### Environment variable

```bash
DATABASE_URL="postgresql://user:password@localhost:5432/mydb?schema=public"
```

## MongoDB + Mongoose _(Express only)_

### Generated files

```text
src/
  lib/
    db.ts              # Mongoose connection helper
  models/
    User.ts            # Example Mongoose model
```

### Connection helper

```typescript
// src/lib/db.ts
import mongoose from "mongoose";

export async function connectDB() {
  try {
    await mongoose.connect(process.env.MONGODB_URI!);
    console.log("MongoDB connected");
  } catch (error) {
    console.error("MongoDB connection error:", error);
    process.exit(1);
  }
}
```

### Mongoose adds no extra scripts

Mongoose does not contribute additional `npm run` scripts. The connection is established by calling `connectDB()` in your application entry point.

### Environment variable

```bash
MONGODB_URI="mongodb://localhost:27017/myapp"
```

## Migration tips

### Prisma

```bash
# Create and apply a migration in development
npx prisma migrate dev --name descriptive_name

# Apply pending migrations in production
npx prisma migrate deploy
```

### Drizzle

```bash
# Generate SQL migration files from schema changes
npm run db:generate

# Run pending migrations
npm run db:migrate
```

### Mongoose

Mongoose applies schema changes at runtime. For production migrations, use a tool such as `migrate-mongo`.

<Callout type="tip">
  Start with **PostgreSQL + Prisma** if you are unsure. It has the broadest
  support across both Express and Next.js, and offers the richest developer
  experience with generated TypeScript types.
</Callout>
