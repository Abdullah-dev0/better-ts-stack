## Overview

The Frontend module generates a Next.js 16 application with React 19 and Tailwind CSS v4. It is automatically selected when you choose the **Full-stack** application type — there is no framework prompt.

<Callout type="info">
  The frontend module produces a **single Next.js project** at the project root.
  There are no separate `frontend/` or `backend/` subdirectories.
</Callout>

## What You Get

```
app/
├── layout.tsx       # Root layout with fonts and metadata
├── page.tsx         # Home page
└── globals.css      # Tailwind CSS v4 import + base styles
public/              # Static assets (Next.js defaults)
proxy.ts             # Dev proxy configuration (rendered from proxy.ts.hbs)
next.config.ts       # Next.js configuration
postcss.config.mjs   # PostCSS setup for Tailwind
tsconfig.json        # TypeScript configuration
eslint.config.mjs    # ESLint configuration
next-env.d.ts        # Next.js TypeScript environment types
```

<Callout type="info">
  The base Next.js template does **not** generate `components/ui`,
  `lib/utils.ts`, or any example API client. Those can be added manually after
  generation.
</Callout>

## Generated Files

### app/layout.tsx

Root layout with Geist fonts and base metadata:

```tsx
import type { Metadata } from "next";
import { Geist, Geist_Mono } from "next/font/google";

import "./globals.css";

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
});

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
});

export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
  children,
}: Readonly<{ children: React.ReactNode }>) {
  return (
    <html lang="en">
      <body
        className={`${geistSans.variable} ${geistMono.variable} antialiased`}
      >
        {children}
      </body>
    </html>
  );
}
```

### app/globals.css

Tailwind CSS v4 setup with system dark-mode support:

```css
@import "tailwindcss";

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --font-sans: var(--font-geist-sans);
  --font-mono: var(--font-geist-mono);
}

:root {
  --background: #ffffff;
  --foreground: #171717;
}

@media (prefers-color-scheme: dark) {
  :root {
    --background: #0a0a0a;
    --foreground: #ededed;
  }
}

body {
  background: var(--background);
  color: var(--foreground);
  font-family: Arial, Helvetica, sans-serif;
}
```

## Integration with Auth Module

If Better Auth is selected (requires a database), the auth module adds:

- `lib/auth.ts` — server-side Better Auth configuration with the matching adapter
- `lib/auth-client.ts` — `createAuthClient()` for use in client components
- `app/api/auth/[...all]/route.ts` — the Better Auth route handler

<Tabs items={['Prisma adapter', 'Drizzle adapter']}>
  <Tab value="Prisma adapter">
    ```typescript
    // lib/auth.ts
    import { betterAuth } from "better-auth";
    import { nextCookies } from "better-auth/next-js";
    import { prismaAdapter } from "better-auth/adapters/prisma";
    import client from "./prisma";

    export const auth = betterAuth({
      database: prismaAdapter(client, { provider: "postgres" }),
      emailAndPassword: { enabled: true },
      plugins: [nextCookies()],
      secret: process.env.BETTER_AUTH_SECRET,
      baseURL: process.env.BETTER_AUTH_URL,
    });
    ```

  </Tab>
  <Tab value="Drizzle adapter">
    ```typescript
    // lib/auth.ts
    import { betterAuth } from "better-auth";
    import { nextCookies } from "better-auth/next-js";
    import { drizzleAdapter } from "better-auth/adapters/drizzle";
    import { db } from "./db";

    export const auth = betterAuth({
      database: drizzleAdapter(db, { provider: "postgres" }),
      emailAndPassword: { enabled: true },
      plugins: [nextCookies()],
      secret: process.env.BETTER_AUTH_SECRET,
      baseURL: process.env.BETTER_AUTH_URL,
    });
    ```

  </Tab>
</Tabs>

### Auth route handler

```typescript
// app/api/auth/[...all]/route.ts
import { toNextJsHandler } from "better-auth/next-js";

import { auth } from "@/lib/auth";

export const { GET, POST } = toNextJsHandler(auth);
```

### Client-side auth

```typescript
// lib/auth-client.ts
import { createAuthClient } from "better-auth/react";

export const authClient = createAuthClient();
```

<Callout type="info">
  `BETTER_AUTH_SECRET` and `BETTER_AUTH_URL` are server-side variables. Do not
  prefix them with `NEXT_PUBLIC_`.
</Callout>

## Extending the Frontend

### Add a page

```tsx
// app/about/page.tsx
export default function AboutPage() {
  return (
    <main>
      <h1>About</h1>
    </main>
  );
}
```

### Add a server-side API route

```tsx
// app/api/hello/route.ts
export async function GET() {
  return Response.json({ message: "Hello from Next.js!" });
}
```

### Add a component

```tsx
// components/Header.tsx
export function Header() {
  return (
    <header className="border-b p-4">
      <nav>Navigation here</nav>
    </header>
  );
}
```

<Callout type="tip">
  Components that use `useState`, `useEffect`, or event handlers must have `"use
  client"` at the top of the file. All other components default to Server
  Components.
</Callout>
