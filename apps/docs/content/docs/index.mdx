## What is better-ts-stack?

better-ts-stack is an interactive CLI tool that generates fully configured TypeScript projects with your choice of backend framework, database, authentication, and Docker support. No config files to edit by hand — just answer a few prompts and get a production-ready codebase.

<Callout type="info" title="Good for">
  - Starting new TypeScript backend or full-stack projects - Prototyping APIs
  with a type-safe Express setup - Learning modern TypeScript project structure
  - Standardizing project scaffolding across a team
</Callout>

## Quick Start

<Steps>

<Step>
### Run the CLI

<Tabs items={['npx (recommended)', 'Global install', 'pnpm dlx', 'bunx']}>
  <Tab value="npx (recommended)">
    ```bash
    npx better-ts-stack
    ```
  </Tab>
  <Tab value="Global install">
    ```bash
    npm install -g better-ts-stack
    better-ts-stack
    ```
  </Tab>
  <Tab value="pnpm dlx">
    ```bash
    pnpm dlx better-ts-stack
    ```
  </Tab>
  <Tab value="bunx">
    ```bash
    bunx better-ts-stack
    ```
  </Tab>
</Tabs>
</Step>

<Step>
### Answer the prompts

The CLI is fully interactive — there are no flags or subcommands. You will be asked:

1. **Project name** — becomes your folder name and `package.json` name
2. **Application type** — `Backend API` or `Full-stack`
   - **Backend**: select a framework (Express now; NestJS coming soon)
   - **Full-stack**: Next.js is auto-selected
3. **Database type** — `none`, `PostgreSQL`, or `MongoDB`
4. **ORM/ODM** (only shown when a database is selected)
   - PostgreSQL → `Prisma` or `Drizzle`
   - MongoDB → `Prisma` or `Mongoose`
5. **Package manager** — `npm`, `pnpm`, or `bun`
6. **Docker** — include Docker configuration?
7. **Authentication** — add auth?
   - Express: JWT-based auth (always available when asked)
   - Next.js: Better Auth (only prompted if a database was selected)
8. **Git** — initialize a repository with an initial commit?
9. **Install dependencies** — run the install command now?

<Callout type="info">
  The CLI does not ask for a port number. The server port defaults to `3000`
  via the `PORT` environment variable.
</Callout>
</Step>

<Step>
### Start developing

```bash
cd your-project-name
npm install   # skip if you chose to install during setup
npm run dev
```

</Step>

</Steps>

## What's Included

<Cards>
  <Card title="Backend API" icon={<Database className="text-blue-400" />}>

    health check endpoint
  </Card>
  <Card title="Full-stack App" icon={<Layout className="text-green-400" />}>
    Next.js 16 with React 19, App Router, Tailwind CSS v4, and a proxy
    configuration
  </Card>
  <Card
    title="Type-safe Database"
    icon={<Shield className="text-purple-400" />}

  </Card>
  <Card title="Authentication" icon={<Zap className="text-yellow-400" />}>

  </Card>
  <Card title="Docker Ready" icon={<Container className="text-blue-400" />}>

  </Card>
  <Card title="Developer Tooling" icon={<Terminal className="text-primary" />}>
    ESLint 9, Prettier, TypeScript strict mode, and hot reload out of the box
  </Card>
</Cards>

## Requirements

- **Node.js** 18.0.0 or higher
- **Package manager** — npm, pnpm, or bun

<Callout type="warn">
  Check your Node.js version before running: `node --version`
</Callout>
