### Generated schema.prisma

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        String   @id @default(uuid())
  email     String   @unique
  password  String
  name      String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

## Handlebars Variables

Templates use these variables:

| Variable | Description | Example |
|----------|-------------|---------|
| `{{projectName}}` | Project name | `my-api` |
| `{{projectNameKebab}}` | Kebab-case name | `my-api` |
| `{{projectNameCamel}}` | CamelCase name | `myApi` |
| `{{database}}` | Selected database | `postgresql` |
| `{{auth}}` | Auth enabled | `true` |
| `{{docker}}` | Docker enabled | `true` |

### Example Template

```typescript
// src/index.ts
import app from './app';

const PORT = process.env.PORT || 3000;
const appName = '{{projectName}}';
```

## Customization

All generated templates are fully editable. After generation, you can:

- Modify any file
- Add new routes/controllers
- Change database schema
- Update configurations
- Extend with additional modules

<Callout type="info">
  The templates provide a solid foundation following best practices, but they're meant to be customized for your specific needs.
</Callout>
